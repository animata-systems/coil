# COIL:ru-mat — неофициальный диалект

> «Если язык можно писать на языке, на котором ты думаешь, — значит, можно и на этом.»

**Статус:** community-диалект, не входит в официальную поставку.

Данный диалект построен на принципе семантического резонанса: каждое ключевое слово не просто заменено матерным аналогом, а подобрано так, чтобы его бытовой смысл отражал суть оператора.

---

## Операторы Core

| COIL | МАТ-диалект | Почему так |
|------|-------------|------------|
| `УЧАСТНИКИ` | `ЕБЛАНЫ` | Объявляешь участников — представляешь команду. «Вот наши ебланы.» |
| `ИНСТРУМЕНТЫ` | `ХУЁВИНЫ` | Инструменты = штуковины. Любой инженер так называет тулы, которым не помнит названий. |
| `ОПРЕДЕЛИ` | `ЗАХУЯЧЬ` | Создать новое значение = захуячить что-то из ничего. Акт творения. |
| `УСТАНОВИ` | `ПЕРЕХУЯЧЬ` | Изменить существующее значение = перехуячить. Было 3, стало 4. Перехуячили. |
| `ПОЛУЧИ` | `СПИЗДИ` | Получить значение из среды = спиздить. Данные сами не придут — надо стащить. |
| `ДУМАЙ` | `ЕБИСЬ` | Когнитивная работа LLM = ебаться с задачей. Точнейшая метафора. |
| `ВЫПОЛНИ` | `ХУЯРЬ` | Запустить инструмент = хуярить. Не думать, а делать. |
| `НАПИШИ` | `ВЫСРИ` | Отправить сообщение = высрать текст в канал. Честное описание процесса. |
| `ЖДИ` | `ЗАЛИПНИ` | Точка синхронизации = залипнуть. Ничего не делаешь, ждёшь промисы. |
| `ВЫХОД` | `СЪЕБИСЬ` | Завершить протокол = съебаться. Сделал дело — съебись. |
| `КОНЕЦ` | `ПИЗДЕЦ` | Закрыть блок. Любой блок заканчивается пиздецом. Как и всё в жизни. |

## Модификаторы

### Оснащение (ДУМАЙ / ВЫПОЛНИ)

| COIL | МАТ-диалект | Почему так |
|------|-------------|------------|
| `ЧЕРЕЗ` | `ЖОПОЙ` | Через какую модель работаем. «ЖОПОЙ $gpt4» — работаем жопой gpt4. Звучит правдоподобно. |
| `КАК` | `РОЖЕЙ` | С какой квалификацией решаем. «РОЖЕЙ $аналитик» — делаем рожу аналитика. |
| `ИСПОЛЬЗУЯ` | `ХУЯЧА` | Какими инструментами. «ХУЯЧА !поиск» — хуяча поиском по базе. |

### Постановка задачи (ДУМАЙ)

| COIL | МАТ-диалект | Почему так |
|------|-------------|------------|
| `ЦЕЛЬ` | `НАХУЯ` | Зачем решаем задачу. Главный вопрос любого проекта наконец задан прямо. |
| `ВХОД` | `ЧО БЛЯТЬ` | Что дано. «ЧО БЛЯТЬ: вот запрос пользователя.» |
| `КОНТЕКСТ` | `ДА ЕЩЁ` | Дополнительные данные. «Да ещё вот это учти.» |
| `РЕЗУЛЬТАТ` | `ВЫСЕРИ` | Что LLM обязана выдать. Structured output = структурированный высер. |

### Адресация (НАПИШИ)

| COIL | МАТ-диалект | Почему так |
|------|-------------|------------|
| `КУДА` | `КУДА НАХУЙ` | Адрес канала. Вечный вопрос: куда нахуй слать? |
| `КОМУ` | `КАКОМУ ХУИЛУ` | Получатель. Адресная доставка. |
| `ОТВЕТ НА` | `НА КАКУЮ ХУЙНЮ` | На какое сообщение отвечаем. |
| `ЖДАТЬ` | `ПОТУПИ` | Политика ожидания ответа. Отправил — потупи. |
| `НЕ БОЛЕЕ` | `ИЛИ ПИЗДЕЦ` | Таймаут. 10 минут — или пиздец. Лучшее описание дедлайна. |

### Ожидание (ЖДИ)

| COIL | МАТ-диалект | Почему так |
|------|-------------|------------|
| `НА` | `ПОКА НЕ РОДЯТ` | Ждём обещания. Как ждём ребёнка — пока не родят. |
| `РЕЖИМ` | `КАК ИМЕННО` | Режим ожидания. |
| `ВСЕ` | `ВСЕХ НАХУЙ` | Ждать все обещания. «КАК ИМЕННО ВСЕХ НАХУЙ» — ждём каждого. |
| `ЛЮБОЙ` | `ДА ПОХУЙ КОГО` | Ждать любое обещание. Первый ответивший побеждает. |

## Extended-операторы

| COIL | МАТ-диалект | Почему так |
|------|-------------|------------|
| `ЕСЛИ` | `А ХУЛИ ЕСЛИ` | Условное ветвление. Наконец if-statement звучит по-человечески. |
| `ПОВТОРЯЙ` | `ДОЛБИСЬ` | Цикл. «ДОЛБИСЬ 5» — повторяй до 5 раз. Точное описание debugging-цикла. |
| `КАЖДЫЙ` | `ОТЫМЕЙ` | Итерация по списку = отыметь каждый элемент. «ОТЫМЕЙ $task ИЗ $plan.files» — обработать каждого. |
| `СОБЕРИ` | `ОГРЕБИ` | Агрегировать результаты = огрести всё в кучу. |
| `СИГНАЛ` | `ПИХНИ` | Отправить данные в поток = пихнуть данные. Мягкий, кооперативный пих. |

## Типы РЕЗУЛЬТАТА

| COIL | МАТ-диалект | Почему так |
|------|-------------|------------|
| `ТЕКСТ` | `ХУЙНЯ` | Строковое значение. Любая строка — хуйня какая-то. |
| `ЧИСЛО` | `ЦИФИРЬ` | Числовое значение. Не мат, но в контексте звучит. |
| `ФЛАГ` | `ЗАЛЁТ` | Булево значение. Залёт или нет. Да/нет. Бинарно. |
| `ВЫБОР(...)` | `ХУЙ ЛИ(...)` | Enum. «ХУЙ ЛИ(баг, фича, вопрос)» — вот и весь выбор. |
| `СПИСОК` | `КУЧАМАЛА` | Массив. Куча-мала элементов. |

## Сигнатуры ссылок

Префиксы не меняются — они часть лексики, а не диалекта:

| Префикс | Значение | Как читать в диалекте |
|----------|----------|----------------------|
| `$` | Значение | «баблишко» (ценность) |
| `?` | Обещание | «залёт» (ожидание результата) |
| `@` | Участник | «еблан» |
| `!` | Инструмент | «хуёвина» |
| `#` | Канал | «дыра» |
| `~` | Поток | «течка» |

## Пример: исследовательский агент

```coil
' ──────────────────────────────────────────────
' Вот наши ебланы и хуёвины
' ──────────────────────────────────────────────
ЕБЛАНЫ эксперт, автор
ХУЁВИНЫ поиск

' ──────────────────────────────────────────────
' Скиллы
' ──────────────────────────────────────────────
ЗАХУЯЧЬ research_skill
<<
Ты — аналитик-исследователь.
Работаешь с неструктурированными запросами.
Всегда ищешь первоисточники и указываешь степень
уверенности в выводах. Избегаешь спекуляций.
Если данных недостаточно — говоришь об этом прямо.
>>
ПИЗДЕЦ

ЗАХУЯЧЬ domain_expertise
<<
Ты разбираешься в продуктовой аналитике.
Знаешь метрики retention, activation, churn.
Умеешь читать воронки и когортные отчёты.
>>
ПИЗДЕЦ

' ──────────────────────────────────────────────
' Входные данные
' ──────────────────────────────────────────────
СПИЗДИ запрос
<<
Запрос пользователя для исследования.
>>
ПИЗДЕЦ

СПИЗДИ история_обращений
<<
История предыдущих обращений по этой теме.
>>
ПИЗДЕЦ

СПИЗДИ кейс
<<
Идентификатор текущего кейса.
>>
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 1. Ебись с анализом
' ──────────────────────────────────────────────
ЕБИСЬ анализ
  РОЖЕЙ $research_skill, $domain_expertise
  НАХУЯ <<
  Подготовить обоснованный ответ с источниками.
  >>
  ЧО БЛЯТЬ <<
  Исследуй запрос пользователя. Учитывай все данные,
  которые поступят в процессе через поток.
  >>
  ДА ЕЩЁ <<
  Запрос: $запрос
  История обращений: $история_обращений
  >>
  ВЫСЕРИ
  * ответ: ХУЙНЯ - итоговый ответ
  * источники: КУЧАМАЛА - использованные материалы
    * ref: ХУЙНЯ - ссылка
    * why: ХУЙНЯ - почему релевантен
  * уверенность: ХУЙ ЛИ(высокая, средняя, низкая)
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 2. Параллельно хуярим поиск и высираем эксперту
' ──────────────────────────────────────────────
ХУЯРЬ найденное
  ХУЯЧА !поиск
  - query: $запрос
  - limit: 10
ПИЗДЕЦ

ВЫСРИ мнение
  КУДА НАХУЙ #консультации
  КАКОМУ ХУИЛУ @эксперт
  ПОТУПИ ДА ПОХУЙ КОГО
  ИЛИ ПИЗДЕЦ 10м
  <<
  Нужна экспертиза по запросу: $запрос
  >>
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 3. Пихаем данные в поток по мере поступления
' ──────────────────────────────────────────────
ДОЛБИСЬ 2
  ЗАЛИПНИ данные
    ПОКА НЕ РОДЯТ ?найденное, ?мнение
    КАК ИМЕННО ДА ПОХУЙ КОГО
  ПИЗДЕЦ

  ПИХНИ ~анализ
    <<
    $данные
    >>
  ПИЗДЕЦ
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 4. Залипаем на финальный результат
' ──────────────────────────────────────────────
ЗАЛИПНИ
  ПОКА НЕ РОДЯТ ?анализ
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 5. Высираем ответ автору
' ──────────────────────────────────────────────
ВЫСРИ
  КУДА НАХУЙ #результаты/$кейс
  КАКОМУ ХУИЛУ @автор
  <<
  $анализ.ответ
  >>
ПИЗДЕЦ

СЪЕБИСЬ
```

## FAQ

**Q: Это шутка?**
A: Нет. Это proof of concept диалектной системы COIL. Тот факт, что скрипт остаётся полностью валидным и читаемым, доказывает, что ключевые слова — это skin, а не skeleton языка.

**Q: Можно использовать в продакшене?**
A: Технически — да. Среда исполнения оперирует семантикой конструкций, а не их написанием. Морально — на ваше усмотрение.

**Q: А другие диалекты?**
A: Точно так же можно построить диалекты на корейском, немецком, суахили или клингонском. Мат-диалект просто нагляднее всего демонстрирует принцип.
