# COIL:ru-mat — неофициальный диалект

> «Если язык можно писать на языке, на котором ты думаешь, — значит, можно и на этом.»

**Статус:** community-диалект, не входит в официальную поставку.

Данный диалект построен на принципе семантического резонанса: каждое ключевое слово не просто заменено матерным аналогом, а подобрано так, чтобы его бытовой смысл отражал суть оператора.

---

## Операторы Core

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Op.Actors` | `ЕБЛАНЫ` | Объявляешь участников — представляешь команду. «Вот наши ебланы.» |
| `Op.Tools` | `ХУЁВИНЫ` | Инструменты = штуковины. Любой инженер так называет тулы, которым не помнит названий. |
| `Op.Define` | `ЗАХУЯЧЬ` | Создать новое значение = захуячить что-то из ничего. Акт творения. |
| `Op.Set` | `ПЕРЕХУЯЧЬ` | Изменить существующее значение = перехуячить. Было 3, стало 4. Перехуячили. |
| `Op.Receive` | `СПИЗДИ` | Получить значение из среды = спиздить. Данные сами не придут — надо стащить. |
| `Op.Think` | `ЕБИСЬ` | Когнитивная работа LLM = ебаться с задачей. Точнейшая метафора. |
| `Op.Execute` | `ХУЯРЬ` | Запустить инструмент = хуярить. Не думать, а делать. |
| `Op.Send` | `ВЫСРИ` | Отправить сообщение = высрать текст в канал. Честное описание процесса. |
| `Op.Wait` | `ЗАЛИПНИ` | Точка синхронизации = залипнуть. Ничего не делаешь, ждёшь промисы. |
| `Op.Exit` | `СЪЕБИСЬ` | Завершить протокол = съебаться. Сделал дело — съебись. |
| `Kw.End` | `ПИЗДЕЦ` | Закрыть блок. Любой блок заканчивается пиздецом. Как и всё в жизни. |

## Модификаторы

### Оснащение (ДУМАЙ / ВЫПОЛНИ)

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Mod.Via` | `ЖОПОЙ` | Через какую модель работаем. «ЖОПОЙ $gpt4» — работаем жопой gpt4. Звучит правдоподобно. |
| `Mod.As` | `РОЖЕЙ` | С какой квалификацией решаем. «РОЖЕЙ $аналитик» — делаем рожу аналитика. |
| `Mod.Using` | `ХУЯЧА` | Какими инструментами. «ХУЯЧА !поиск» — хуяча поиском по базе. |

### Постановка задачи (ДУМАЙ)

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Mod.Goal` | `НАХУЯ` | Зачем решаем задачу. Главный вопрос любого проекта наконец задан прямо. |
| `Mod.Input` | `ЧО БЛЯТЬ` | Что дано. «ЧО БЛЯТЬ: вот запрос пользователя.» |
| `Mod.Context` | `ДА ЕЩЁ` | Дополнительные данные. «Да ещё вот это учти.» |
| `Mod.Result` | `ВЫСЕРИ` | Что LLM обязана выдать. Structured output = структурированный высер. |

### Адресация (НАПИШИ)

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Mod.To` | `КУДА НАХУЙ` | Адрес канала. Вечный вопрос: куда нахуй слать? |
| `Mod.For` | `КАКОМУ ХУИЛУ` | Получатель. Адресная доставка. |
| `Mod.ReplyTo` | `НА КАКУЮ ХУЙНЮ` | На какое сообщение отвечаем. |
| `Mod.Await` | `ПОТУПИ` | Политика ожидания ответа. Отправил — потупи. |
| `Mod.Timeout` | `ИЛИ ПИЗДЕЦ` | Таймаут. 10 минут — или пиздец. Лучшее описание дедлайна. |

### Ожидание (ЖДИ)

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Mod.On` | `ПОКА НЕ РОДЯТ` | Ждём обещания. Как ждём ребёнка — пока не родят. |
| `Mod.Mode` | `КАК ИМЕННО` | Режим ожидания. |
| `Pol.All` | `ВСЕХ НАХУЙ` | Ждать все обещания. «КАК ИМЕННО ВСЕХ НАХУЙ» — ждём каждого. |
| `Pol.Any` | `ДА ПОХУЙ КОГО` | Ждать любое обещание. Первый ответивший побеждает. |

## Extended-операторы

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Op.If` | `А ХУЛИ ЕСЛИ` | Условное ветвление. Наконец if-statement звучит по-человечески. |
| `Op.Repeat` | `ДОЛБИСЬ` | Цикл. «ДОЛБИСЬ 5» — повторяй до 5 раз. Точное описание debugging-цикла. |
| `Op.Each` | `ОТЫМЕЙ` | Итерация по списку = отыметь каждый элемент. «ОТЫМЕЙ $task ИЗ $plan.files» — обработать каждого. |
| `Op.Gather` | `ОГРЕБИ` | Агрегировать результаты = огрести всё в кучу. |
| `Op.Signal` | `ПИХНИ` | Отправить данные в поток = пихнуть данные. Мягкий, кооперативный пих. |

### Итерация (ДОЛБИСЬ / ОТЫМЕЙ)

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Mod.Until` | `ПОКА НЕ` | Условие выхода из цикла. «ДОЛБИСЬ ПОКА НЕ $готово ХВАТИТ 5» — долбись, пока не станет готово. |
| `Mod.Limit` | `ХВАТИТ` | Лимит итераций. «ХВАТИТ 5» — хватит, больше пяти раз не надо. Голос разума в бесконечном цикле. |
| `Mod.From` | `ИЗ` | Источник списка. «ОТЫМЕЙ $task ИЗ $plan.files» — взять из списка и обработать. |
| `Pol.None` | `НАСРАТЬ` | Не ждать ответа. Fire and forget. «ПОТУПИ НАСРАТЬ» — отправил и насрать, ответят или нет. |

## Типы РЕЗУЛЬТАТА

| ID | МАТ-диалект | Почему так |
|------|-------------|------------|
| `Typ.Text` | `ХУЙНЯ` | Строковое значение. Любая строка — хуйня какая-то. |
| `Typ.Number` | `ЦИФИРЬ` | Числовое значение. Не мат, но в контексте звучит. |
| `Typ.Flag` | `ЗАЛЁТ` | Булево значение. Залёт или нет. Да/нет. Бинарно. |
| `Typ.Choice` | `ХУЙ ЛИ(...)` | Enum. «ХУЙ ЛИ(баг, фича, вопрос)» — вот и весь выбор. |
| `Typ.List` | `КУЧАМАЛА` | Массив. Куча-мала элементов. |

## Сигнатуры ссылок

Префиксы не меняются — они часть лексики, а не диалекта:

| Префикс | Значение | Как читать в диалекте |
|----------|----------|----------------------|
| `$` | Значение | «баблишко» (ценность) |
| `?` | Обещание | «залёт» (ожидание результата) |
| `@` | Участник | «еблан» |
| `!` | Инструмент | «хуёвина» |
| `#` | Канал | «дыра» |
| `~` | Поток | «течка» |

## Пример: исследовательский агент

```coil
' ──────────────────────────────────────────────
' Вот наши ебланы и хуёвины
' ──────────────────────────────────────────────
ЕБЛАНЫ эксперт, автор
ХУЁВИНЫ поиск

' ──────────────────────────────────────────────
' Скиллы
' ──────────────────────────────────────────────
ЗАХУЯЧЬ research_skill
<<
Ты — аналитик-исследователь.
Работаешь с неструктурированными запросами.
Всегда ищешь первоисточники и указываешь степень
уверенности в выводах. Избегаешь спекуляций.
Если данных недостаточно — говоришь об этом прямо.
>>
ПИЗДЕЦ

ЗАХУЯЧЬ domain_expertise
<<
Ты разбираешься в продуктовой аналитике.
Знаешь метрики retention, activation, churn.
Умеешь читать воронки и когортные отчёты.
>>
ПИЗДЕЦ

' ──────────────────────────────────────────────
' Входные данные
' ──────────────────────────────────────────────
СПИЗДИ запрос
<<
Запрос пользователя для исследования.
>>
ПИЗДЕЦ

СПИЗДИ история_обращений
<<
История предыдущих обращений по этой теме.
>>
ПИЗДЕЦ

СПИЗДИ кейс
<<
Идентификатор текущего кейса.
>>
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 1. Ебись с анализом
' ──────────────────────────────────────────────
ЕБИСЬ анализ
  РОЖЕЙ $research_skill, $domain_expertise
  НАХУЯ <<
  Подготовить обоснованный ответ с источниками.
  >>
  ЧО БЛЯТЬ <<
  Исследуй запрос пользователя. Учитывай все данные,
  которые поступят в процессе через поток.
  >>
  ДА ЕЩЁ <<
  Запрос: $запрос
  История обращений: $история_обращений
  >>
  ВЫСЕРИ
  * ответ: ХУЙНЯ - итоговый ответ
  * источники: КУЧАМАЛА - использованные материалы
    * ref: ХУЙНЯ - ссылка
    * why: ХУЙНЯ - почему релевантен
  * уверенность: ХУЙ ЛИ(высокая, средняя, низкая)
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 2. Параллельно хуярим поиск и высираем эксперту
' ──────────────────────────────────────────────
ХУЯРЬ найденное
  ХУЯЧА !поиск
  - query: $запрос
  - limit: 10
ПИЗДЕЦ

ВЫСРИ мнение
  КУДА НАХУЙ #консультации
  КАКОМУ ХУИЛУ @эксперт
  ПОТУПИ ДА ПОХУЙ КОГО
  ИЛИ ПИЗДЕЦ 10м
  <<
  Нужна экспертиза по запросу: $запрос
  >>
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 3. Пихаем данные в поток по мере поступления
' ──────────────────────────────────────────────
ДОЛБИСЬ 2
  ЗАЛИПНИ данные
    ПОКА НЕ РОДЯТ ?найденное, ?мнение
    КАК ИМЕННО ДА ПОХУЙ КОГО
  ПИЗДЕЦ

  ПИХНИ ~анализ
    <<
    $данные
    >>
  ПИЗДЕЦ
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 4. Залипаем на финальный результат
' ──────────────────────────────────────────────
ЗАЛИПНИ
  ПОКА НЕ РОДЯТ ?анализ
ПИЗДЕЦ

' ──────────────────────────────────────────────
' 5. Высираем ответ автору
' ──────────────────────────────────────────────
ВЫСРИ
  КУДА НАХУЙ #результаты/$кейс
  КАКОМУ ХУИЛУ @автор
  <<
  $анализ.ответ
  >>
ПИЗДЕЦ

СЪЕБИСЬ
```

## FAQ

**Q: Это шутка?**
A: Нет. Это proof of concept диалектной системы COIL. Тот факт, что скрипт остаётся полностью валидным и читаемым, доказывает, что ключевые слова — это skin, а не skeleton языка.

**Q: Можно использовать в продакшене?**
A: Технически — да. Среда исполнения оперирует семантикой конструкций, а не их написанием. Морально — на ваше усмотрение.

**Q: А другие диалекты?**
A: Точно так же можно построить диалекты на корейском, немецком, суахили или клингонском. Мат-диалект просто нагляднее всего демонстрирует принцип.
