# COIL — Cognitive Orchestration Interface Language

**Язык сценариев для когнитивной оркестрации.**

## Зачем

Разрабатывается операционная система нового поколения, где в роли процессора выступает LLM. Каждому агенту в такой ОС нужен сценарий поведения — не промпт («кто ты»), а протокол: что делать, когда ждать, кого спросить, куда написать.

COIL — язык для описания таких протоколов. Его читают и пишут одновременно человек и LLM. Человек видит понятный сценарий на своём языке. LLM видит однозначную структуру, которую может исполнять, генерировать и улучшать.

## Манифест

1. **COIL ставит пользователя в центр.** Язык должен оставаться читаемым и доступным не-программисту. Ключевые слова пишутся на естественном языке пользователя.

2. **COIL управляет, а не программирует.** Задача языка — описывать ход работы, а не вычислительные алгоритмы общего назначения.

3. **COIL делает логику явной.** Ветвления, ожидания, обращения, вызовы и завершение должны быть видны в тексте сценария.

4. **COIL разделяет жёсткое и мягкое.** Управляющий каркас исполняется детерминированно. LLM вызывается только там, где действительно нужна когнитивная работа.

5. **COIL рождён для агентной ОС.** Он говорит в терминах участников, каналов, сообщений, инструментов, потоков и событий.

## Hello World

Минимальный сценарий — отправить сообщение в канал:

```coil
НАПИШИ
  КУДА #general
  <<
  Hello, world!
  >>
КОНЕЦ

ВЫХОД
```

Одна команда, один канал, ноль LLM. Агент отправляет сообщение и завершается.

## Два представления одного языка

Сценарии COIL пишут и люди, и LLM. У них разные потребности. Вместо компромисса язык имеет две проекции с единой семантикой:

**COIL-C** *(canonical)* — текстовая машинная форма. Verbose, явная, однозначная. Оптимизирована для генерации LLM и машинного парсинга.

**COIL-H** *(human)* — табличная проекция, похожая на пронумерованную спецификацию. Оптимизирована для чтения и редактирования человеком в COIL IDE.

COIL-H транслируется в COIL-C без потерь смысла. Семантика живёт в COIL-C.

Тот же Hello World — теперь с когнитивным шагом:

## Мультиязычность

Среда исполнения оперирует семантикой конструкций, а не их написанием. Ключевые слова пишутся на том языке, на котором оператор думает. Между мыслью и текстом сценария не должно быть перевода.

Один и тот же сценарий на разных языках — это один и тот же протокол:

```coil
ДУМАЙ анализ                          THINK analysis
  ЦЕЛЬ <<                               GOAL <<
  Классифицируй запрос.                 Classify the request.
  >>                                    >>
  РЕЗУЛЬТАТ                             RESULT
  * тип: ВЫБОР(баг, фича, вопрос)       * type: CHOICE(bug, feature, question)
КОНЕЦ                                 END
```

COIL IDE переводит всё автоматически — операторы, модификаторы, содержимое шаблонов, комментарии.

## Ключевые идеи

**Жёсткое и мягкое.** Управляющий каркас (порядок шагов, ожидание, адресация, таймауты) исполняется детерминированно, без LLM. Содержимое шаблонов `<< >>` передаётся LLM или участнику как есть. Два режима не смешиваются.

**Типизированные ссылки.** Шесть префиксов — шесть типов сущностей. Каждый символ однозначно сообщает, с чем имеешь дело:

| Префикс | Что это                  | Пример |
|---|--------------------------|---|
| `$` | Значение                 | `$запрос`, `$план.вывод` |
| `?` | Обещание результата      | `?анализ` |
| `@` | Участник                 | `@эксперт` |
| `!` | Инструмент               | `!поиск` |
| `#` | Адрес (канал, сообщение) | `#результаты/$кейс` |
| `~` | Двусторонний поток       | `~анализ` |

**Постановка задачи для LLM.** Оператор `ДУМАЙ` структурирован как математическая задача: КАК (квалификация решателя), ЦЕЛЬ (зачем), ВХОД (условие), КОНТЕКСТ (дано), РЕЗУЛЬТАТ (найти). LLM выполняет роль решателя.

**Обещания и потоки.** Запускающие операторы (`ДУМАЙ`, `ВЫПОЛНИ`, `НАПИШИ`) создают обещание результата `?name` и двусторонний поток `~name`. Сценарий продолжает выполнение, не дожидаясь ответа. Ожидание — явное, через оператор `ЖДИ`.
