## COIL-C

```coil
' ──────────────────────────────────────────────
' Среда
' ──────────────────────────────────────────────
УЧАСТНИКИ эксперт, автор
ИНСТРУМЕНТЫ поиск

' ──────────────────────────────────────────────
' Скиллы — квалификация решателя.
' Определяют, какими знаниями и подходом
' должен обладать LLM при решении задачи.
' ──────────────────────────────────────────────
ОПРЕДЕЛИ скилл_исследователя
<<
Ты — аналитик-исследователь.
Работаешь с неструктурированными запросами.
Всегда ищешь первоисточники и указываешь степень
уверенности в выводах. Избегаешь спекуляций.
Если данных недостаточно — говоришь об этом прямо.
>>
КОНЕЦ

ОПРЕДЕЛИ доменная_экспертиза
<<
Ты разбираешься в продуктовой аналитике.
Знаешь метрики retention, activation, churn.
Умеешь читать воронки и когортные отчёты.
При анализе всегда разделяешь корреляцию
и причинно-следственную связь.
>>
КОНЕЦ

' ──────────────────────────────────────────────
' Входные данные — среда обязана предоставить
' значения до продолжения протокола.
' ──────────────────────────────────────────────
ПОЛУЧИ запрос
<<
Запрос пользователя для исследования.
>>
КОНЕЦ

ПОЛУЧИ история_обращений
<<
История предыдущих обращений по этой теме.
>>
КОНЕЦ

ПОЛУЧИ кейс
<<
Идентификатор текущего кейса.
>>
КОНЕЦ

' ──────────────────────────────────────────────
' 1. Постановка задачи.
'    ДУМАЙ работает в фоне. Создаёт обещание
'    результата ?анализ и живой поток ~анализ.
'    Условие задачи можно дополнять по ходу
'    решения через СИГНАЛ.
' ──────────────────────────────────────────────
ДУМАЙ анализ
  КАК $скилл_исследователя, $доменная_экспертиза
  ЦЕЛЬ <<
  Подготовить обоснованный ответ с источниками.
  >>
  ВХОД <<
  Исследуй запрос пользователя. Учитывай все данные,
  которые поступят в процессе через поток.
  >>
  КОНТЕКСТ <<
  Запрос: $запрос
  История обращений: $история_обращений
  >>
  РЕЗУЛЬТАТ
  * ответ: ТЕКСТ - итоговый ответ
  * источники: СПИСОК - использованные материалы
    * ref: ТЕКСТ - ссылка
    * why: ТЕКСТ - почему релевантен
  * уверенность: ВЫБОР(высокая, средняя, низкая)
КОНЕЦ

' ──────────────────────────────────────────────
' 2. Сбор данных — параллельно с размышлением.
'    Поиск и запрос эксперту не блокируют
'    ни друг друга, ни ДУМАЙ.
' ──────────────────────────────────────────────
ВЫПОЛНИ найденное
  ИСПОЛЬЗУЯ !поиск
  - query: $запрос
  - limit: 10
КОНЕЦ

НАПИШИ мнение
  КУДА #консультации
  КОМУ @эксперт
  ЖДАТЬ ЛЮБОЙ
  НЕ БОЛЕЕ 10м
  <<
  Нужна экспертиза по запросу: $запрос
  >>
КОНЕЦ

' ──────────────────────────────────────────────
' 3. Дополнение условия по ходу решения.
'    Данные приходят в произвольном порядке —
'    каждый результат сразу уходит в ~анализ.
'    LLM учитывает их, не начиная заново.
' ──────────────────────────────────────────────
ПОВТОРЯЙ 2
  ЖДИ данные
    НА ?найденное, ?мнение
    РЕЖИМ ЛЮБОЙ
  КОНЕЦ

  СИГНАЛ ~анализ
    <<
    $данные
    >>
  КОНЕЦ
КОНЕЦ

' ──────────────────────────────────────────────
' 4. Решение готово.
'    Анализ учёл исходные данные, поиск
'    и мнение эксперта.
' ──────────────────────────────────────────────
ЖДИ
  НА ?анализ
КОНЕЦ

' ──────────────────────────────────────────────
' 5. Ответ автору запроса.
' ──────────────────────────────────────────────
НАПИШИ
  КУДА #результаты/$кейс
  КОМУ @автор
  <<
  $анализ.ответ
  >>
КОНЕЦ

ВЫХОД
```
