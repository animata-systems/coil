# 03. Extended

| | |
|---|---|
| **Статус** | Смешанный: stable surface + experimental semantics (см. ниже) |
| **Назначение** | Расширения, вписанные в проектную модель |

---

## 3.1. Общий статус

Операторы Extended имеют разный нормативный статус:

| Оператор | Surface syntax | Семантика |
|---|---|---|
| `ЕСЛИ` | Stable | Condition slot stable; expression grammar experimental |
| `ПОВТОРЯЙ` | Stable | Обе канонические формы stable; expression grammar experimental |
| `КАЖДЫЙ` | Stable | Stable |
| `СОБЕРИ` | Deferred | Deferred — сознательно не нормализуется в текущем цикле |
| `СИГНАЛ` | Stable | Experimental — зависит от stream semantics |

**Stable** означает: surface syntax заморожен, docs и examples могут этому учить, будущие истории вправе на это опираться.

**Experimental** означает: идея принята, но синтаксис или семантика ещё подвижны.

**Deferred** означает: область признана важной, но сознательно не нормализуется в текущем цикле.

---

## 3.2. `ЕСЛИ`

| | |
|---|---|
| **Surface syntax** | Stable |
| **Expression grammar** | Experimental |

### Назначение

Условное ветвление. Управляет потоком исполнения на основе детерминированно вычисляемого условия.

### Каноническая форма

```coil
ЕСЛИ $score > 80
  ...
КОНЕЦ
```

### Семантика

- Условие вычисляется средой исполнения детерминированно, без вызова LLM.
- `ЕСЛИ` управляет потоком исполнения, а не формулировкой промпта.

### Правила валидности

- Условие должно быть выражением, вычисляемым средой исполнения детерминированно, без вызова LLM.
- Наличие condition slot — stable. Точный синтаксис выражений (операторы сравнения, логические связки, precedence) — experimental (см. [10-terminology.md § 10.10](10-terminology.md)).

---

## 3.3. `ПОВТОРЯЙ`

| | |
|---|---|
| **Surface syntax** | Stable |
| **Expression grammar** | Experimental |

### Назначение

Цикл с обязательным ограничением сверху.

### Канонические формы

Count-only форма:

```coil
ПОВТОРЯЙ 5
  ...
КОНЕЦ
```

Условная форма:

```coil
ПОВТОРЯЙ ДО $готово НЕ БОЛЕЕ 5
  ...
КОНЕЦ
```

### Семантика

- Тело цикла исполняется повторно, пока не достигнут лимит (count-only форма) или пока условие не выполнено либо не достигнут лимит (условная форма).
- Ограничение числа повторов — **часть конструкции**, а не внешняя договорённость.
- Обе формы являются каноническими. Count-only форма — не послабление, а полноценная конструкция.

### Правила валидности

- Лимит повторов обязателен. `ПОВТОРЯЙ` без лимита считается невалидным.
- Точный синтаксис условия в форме `ДО` — experimental (см. [10-terminology.md § 10.10](10-terminology.md)).

## 3.4. `КАЖДЫЙ`

| | |
|---|---|
| **Статус** | Stable |

### Назначение

Итерация по элементам списка. Выполняет тело для каждого элемента.

### Рабочая форма

```coil
КАЖДЫЙ $task ИЗ $plan.files
  ...
КОНЕЦ
```

### Модификаторы

| ID | Ключевое слово (RU) | Назначение | Принимает | Обязательность |
|---|---|---|---|---|
| `Mod.From` | `ИЗ` | Источник — список для итерации | Ссылка `$name` | Да |

### Семантика

- Переменная элемента указывается в сигнатуре как `$name`.
- `ИЗ` принимает ссылку на значение типа `СПИСОК`.
- Тело исполняется последовательно для каждого элемента.
- Итерация детерминирована: управление потоком без LLM.
- Запускающие операторы внутри тела (`ДУМАЙ`, `ВЫПОЛНИ`, `НАПИШИ`) создают обещания на каждой итерации. Среда исполнения вправе исполнять итерации параллельно, если между ними нет зависимостей по данным.

### Правила валидности

- `ИЗ` обязателен.
- Источник должен быть значением типа `СПИСОК`.
- Переменная элемента не должна совпадать с уже определённым именем.

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Источник не является списком | Ошибка исполнения |
| `ИЗ` опущен | Ошибка подготовки |
| Имя переменной конфликтует | Ошибка подготовки |

## 3.5. `СОБЕРИ`

| | |
|---|---|
| **Статус** | Deferred |

### Назначение

Агрегация набора результатов в одно итоговое значение.

> `СОБЕРИ` сознательно не нормализуется в текущем цикле. Точный синтаксис не зафиксирован.

---

## 3.6. `СИГНАЛ`

| | |
|---|---|
| **Surface syntax** | Stable |
| **Семантика** | Experimental — зависит от stream semantics |

### Назначение

Отправка данных в уже существующий поток. Не открывает новый поток.

### Каноническая форма

```coil
СИГНАЛ ~решение
<<
Клиент сообщил новый номер договора: $contract_id
>>
КОНЕЦ
```

### Семантика

- Адресат — уже существующий поток `~name`.
- Доставка кооперативная: поток принимает сигнал в согласованной точке исполнения, а не как hard-interrupt.

---

## 3.7. Устаревшие формы

| Оператор | Статус | Замена |
|---|---|---|
| `ВЫЗОВИ` | Исключён | Объединён с `ВЫПОЛНИ` (Core). One-shot вызов — вырожденный случай `ВЫПОЛНИ` |
| `СПРОСИ` | Исключён | Роль поглощена `НАПИШИ` с модификатором `ЖДАТЬ` |
| `ОТВЕТЬ` | Исключён | Сообщения — `НАПИШИ`, завершение — `ВЫХОД` |
| `КАЖДЫЙ` | Восстановлен | Перенесён обратно в Extended (§ 3.4) |
| `СВЕДИ` | Исключён | Заменяется `СОБЕРИ` (если понадобится) |
