## 2.1. Состав Core

### Сводная таблица операторов

| ID | Ключевое слово (RU) | Категория | Поведение | Создаёт |
|---|---|---|---|---|
| `Op.Actors` | `УЧАСТНИКИ` | Мгновенный | Декларация участников | — |
| `Op.Tools` | `ИНСТРУМЕНТЫ` | Мгновенный | Декларация инструментов | — |
| `Op.Define` | `ОПРЕДЕЛИ` | Мгновенный | Создание нового значения | `$name` |
| `Op.Set` | `УСТАНОВИ` | Мгновенный | Изменение существующего значения | — |
| `Op.Receive` | `ПОЛУЧИ` | Блокирующий | Связывание обязательного значения из среды | `$name` |
| `Op.Wait` | `ЖДИ` | Блокирующий | Ожидание обещаний или событий | — |
| `Op.Think` | `ДУМАЙ` | Запускающий | Когнитивный шаг LLM | `?name`, `$name` (после ожидания) |
| `Op.Execute` | `ВЫПОЛНИ` | Запускающий | Выполнение задачи через инструмент | `?name`, `~name`, `$name` (после ожидания) |
| `Op.Send` | `НАПИШИ` | Запускающий | Отправка сообщения | `?name`, `$name` (после ожидания) |
| `Op.Exit` | `ВЫХОД` | Завершающий | Завершение протокола | — |

**Мгновенные** — среда исполнения обязана выполнить без ожидания и без вызова внешних сторон.

**Блокирующие** — приостанавливают текущий экземпляр протокола до получения результата.

**Запускающие** — инициируют асинхронную работу и создают обещание `?name`.

**Завершающие** — завершают текущий экземпляр протокола.

## 2.2. `УЧАСТНИКИ`

### Назначение

Объявляет имена участников, которые хост-среда обязана предоставить.

### Каноническая форма

```coil
УЧАСТНИКИ
  тех_аналитик
  клиент_а
КОНЕЦ
```

### Допустимые формы

Inline-форма:

```coil
УЧАСТНИКИ тех_аналитик, клиент_а
```

### Семантика

- Регистрирует имена в пространстве участников текущего протокола.
- После объявления участники адресуются как `@тех_аналитик`, `@клиент_а`.
- Не является присваиванием значения и не запускает действие.

### Правила валидности

- Каждое имя должно быть уникальным идентификатором в пределах декларации.
- Использование `@name` без предварительного объявления в `УЧАСТНИКИ` считается ошибкой подготовки.

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Дублирование имени | Ошибка подготовки |
| `@name` без объявления | Ошибка подготовки |
| Хост-среда не может предоставить участника | Ошибка исполнения |

## 2.4. `ОПРЕДЕЛИ`

### Назначение

Создаёт новое именованное значение.

### Каноническая форма

```coil
ОПРЕДЕЛИ retries
3
КОНЕЦ
```

```coil
ОПРЕДЕЛИ analyst_role
<<
Ты аналитик, который ищет артефакты, требующие изменений.
>>
КОНЕЦ
```

Ссылка (алиасинг):

```coil
ОПРЕДЕЛИ current_role
$general_role
КОНЕЦ
```

### Семантика

- Создаёт новое значение `$name` в текущем протоколе.
- Тело оператора — литерал, ссылка `$name` или шаблон `<< >>`.
- Ссылка `$name` в теле означает копирование значения с сохранением типа (алиасинг).

### Результаты

| Сущность | Ссылка |
|---|---|
| Значение | `$name` |

### Правила валидности

- Имя в сигнатуре — голый идентификатор (без префикса).
- Повторное `ОПРЕДЕЛИ` с тем же именем считается ошибкой подготовки. Для изменения существующего значения — `УСТАНОВИ`.

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Имя уже определено | Ошибка подготовки |

## 2.5. `УСТАНОВИ`

### Назначение

Изменяет уже существующее значение.

### Каноническая форма

```coil
УСТАНОВИ $retries
4
КОНЕЦ
```

Ссылка (копирование значения):

```coil
УСТАНОВИ $current_translation
$improved_translation.text
КОНЕЦ
```

### Семантика

- Заменяет значение, привязанное к `$name`.
- В сигнатуре — типизированная ссылка `$name` (значение уже существует).
- Тело оператора — литерал, ссылка `$name` или шаблон `<< >>`. Ссылка в теле означает копирование значения с сохранением типа.

### Правила валидности

- Имя должно быть ранее определено через `ОПРЕДЕЛИ`, `ПОЛУЧИ` или как результат завершённого обещания.
- `УСТАНОВИ` для несуществующего имени считается ошибкой подготовки.

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Имя не определено | Ошибка подготовки |

## 2.6. `ПОЛУЧИ`

### Назначение

Блокирующий шаг, связывающий обязательное значение из среды.

### Каноническая форма

```coil
ПОЛУЧИ contract_id
<<
Укажи номер договора.
>>
КОНЕЦ
```

### Семантика

- Текущий экземпляр протокола не продолжается, пока значение не будет получено.
- Среда исполнения вправе подставить значение мгновенно, запросить его у пользователя или получить из конфигурации хост-среды.
- Это **блокирующее связывание**, а не асинхронная orchestration-операция.

### Результаты

| Сущность | Ссылка |
|---|---|
| Значение | `$name` |

### Условия завершения

Шаг считается завершённым, когда среда исполнения предоставила значение.

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Среда не может предоставить значение | Ошибка исполнения |
| Таймаут (если задан) | Ошибка исполнения |

## 2.7. `ДУМАЙ`

### Назначение

Запускает когнитивный шаг LLM. Структура `ДУМАЙ` — постановка задачи.

### Каноническая форма

```coil
ДУМАЙ plan
  ЧЕРЕЗ $fast_model
  КАК $analyst_role, $troubleshooting
  ИСПОЛЬЗУЯ !search, !calculate
  ЦЕЛЬ <<
  Определи, какие артефакты требуют изменений.
  >>
  ВХОД <<
  Запрос пользователя:
  $request
  >>
  КОНТЕКСТ <<
  История:
  $history
  >>
  РЕЗУЛЬТАТ
  * summary: ТЕКСТ - краткий общий вывод
  * artifacts: СПИСОК - артефакты, требующие изменений
    * path: ТЕКСТ - путь к артефакту
    * action: ВЫБОР(создание, изменение, удаление) - тип изменения
    * reason: ТЕКСТ - почему требуется изменение
КОНЕЦ
```

### Модификаторы

Модификаторы делятся на две группы. Оснащение должно предшествовать постановке.

**Оснащение** — определяет, как и чем исполняется когнитивная работа:

| ID | Ключевое слово (RU) | Назначение | Принимает | Обязательность |
|---|---|---|---|---|
| `Mod.Via` | `ЧЕРЕЗ` | Модель LLM | Ровно одна ссылка `$name` | Нет |
| `Mod.As` | `КАК` | Квалификация решателя (скиллы) | Список ссылок `$name` через `,` | Нет |
| `Mod.Using` | `ИСПОЛЬЗУЯ` | Доступные инструменты | Список ссылок `!name` через `,` | Нет |

**Постановка** — определяет, что решать:

| ID | Ключевое слово (RU) | Назначение | Форма | Обязательность |
|---|---|---|---|---|
| `Mod.Goal` | `ЦЕЛЬ` | Зачем решаем | Шаблон `<< >>` | Нет |
| `Mod.Input` | `ВХОД` | Условие задачи | Шаблон `<< >>` | Нет |
| `Mod.Context` | `КОНТЕКСТ` | Дополнительные данные | Шаблон `<< >>` | Нет |
| `Mod.Result` | `РЕЗУЛЬТАТ` | Что LLM должна определить | Структурный микросинтаксис | Нет |

### Семантика

- LLM выполняет роль решателя: получает постановку и выдаёт структурированный ответ.
- Если указан `ЧЕРЕЗ`, среда исполнения обязана разрешить ссылку и использовать указанную модель. Если модификатор опущен, среда исполнения должна использовать модель по умолчанию.
- Если указан `КАК`, среда исполнения обязана разрешить все ссылки и передать их содержимое как квалификацию решателя.
- Если указан `ИСПОЛЬЗУЯ`, среда исполнения обязана предоставить перечисленные инструменты LLM. LLM вправе вызвать любой из них ноль или более раз. Каждый вызов расходует бюджет.

### Результаты

| Сущность | Ссылка | Когда доступен |
|---|---|---|
| Обещание | `?name` | Сразу после запуска |
| Значение | `$name` | После успешного ожидания |

> Примечание: потоковая проекция `~name` описана в [07-streams.md](07-streams.md) и считается рабочей гипотезой.

### Условия завершения

Шаг считается завершённым, когда LLM вернула результат, прошедший валидацию (включая repair при необходимости).

### Правила валидности

- `ЧЕРЕЗ` принимает ровно одну ссылку `$name`. Литералы **запрещены** — модель должна быть определена через `ОПРЕДЕЛИ` или `ПОЛУЧИ`.
- `КАК` принимает ссылки `$name`. Шаблон `<< >>` **запрещён**.
- `ИСПОЛЬЗУЯ` принимает ссылки `!name`. Все инструменты должны быть объявлены через `ИНСТРУМЕНТЫ`.
- `РЕЗУЛЬТАТ` должен быть последним модификатором. `РЕЗУЛЬТАТ` не закрывается отдельным `КОНЕЦ`.
- Модификаторы оснащения должны предшествовать модификаторам постановки.
- Минимальный `ДУМАЙ` содержит только постановку (хотя бы один модификатор постановки или анонимное тело).

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Неразрешимая ссылка в `КАК` | Ошибка исполнения |
| Неразрешимая ссылка в `ЧЕРЕЗ` | Ошибка исполнения |
| Необъявленный инструмент в `ИСПОЛЬЗУЯ` | Ошибка подготовки |
| Модификатор постановки перед оснащением | Ошибка подготовки |
| Невалидный результат после repair | Ошибка исполнения |

### Пример: минимальный `ДУМАЙ`

```coil
ДУМАЙ решение
  ЦЕЛЬ <<
  Классифицируй запрос клиента.
  >>
  РЕЗУЛЬТАТ
  * категория: ВЫБОР(баг, фича, вопрос) - тип запроса
КОНЕЦ
```

### Невалидный пример

```coil
' ОШИБКА: ЦЕЛЬ (постановка) перед КАК (оснащение)
ДУМАЙ plan
  ЦЕЛЬ <<
  Найди проблему.
  >>
  КАК $analyst_role
КОНЕЦ
```

## 2.8. `ВЫПОЛНИ`

### Назначение

Единый оператор работы с инструментами. Покрывает как одноразовые вызовы, так и интерактивные сессии.

### Каноническая форма

```coil
ВЫПОЛНИ article
  ИСПОЛЬЗУЯ !загрузить_статью
  - url: $url
  - format: "markdown"
КОНЕЦ
```

```coil
ВЫПОЛНИ диалог
  ИСПОЛЬЗУЯ !открыть_чат_с_клиентом
  - contract_id: $contract_id
  - topic: "уточнение времени сбоя"
КОНЕЦ
```

### Модификаторы

| ID | Ключевое слово (RU) | Назначение | Принимает | Обязательность |
|---|---|---|---|---|
| `Mod.Using` | `ИСПОЛЬЗУЯ` | Вызываемый инструмент | Ровно одна ссылка `!name` | Да |

### Аргументы

Аргументы инструмента передаются списком `- key: value` после модификатора `ИСПОЛЬЗУЯ`. Значения — литералы или ссылки `$name`.

> Примечание: `ВЫПОЛНИ` **не принимает** шаблонное тело `<< >>`. 

### Семантика

- Среда исполнения обязана разрешить ссылку `!name`, передать аргументы и запустить работу инструмента.
- `ВЫПОЛНИ` всегда создаёт обещание `?name` и поток `~name`. Для one-shot инструмента поток закрывается сразу после получения результата — это вырожденный случай.
- Контракт аргументов и результата определяется хост-средой и самим инструментом.

### Результаты

| Сущность | Ссылка | Когда доступен |
|---|---|---|
| Обещание | `?name` | Сразу после запуска |
| Поток | `~name` | Сразу после запуска |
| Значение | `$name` | После успешного ожидания |

### Условия завершения

Шаг считается завершённым, когда инструмент вернул финальный результат. Для интерактивных сессий — когда сессия закрыта.

### Правила валидности

- `ИСПОЛЬЗУЯ` обязателен и принимает ровно одну ссылку `!name`.
- Инструмент должен быть объявлен через `ИНСТРУМЕНТЫ`.
- Шаблонное тело `<< >>` запрещено. Наличие `<< >>` в блоке `ВЫПОЛНИ` считается ошибкой подготовки.

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Необъявленный инструмент | Ошибка подготовки |
| Шаблонное тело `<< >>` в блоке | Ошибка подготовки |
| Отказ инструмента | Ошибка исполнения |
| Таймаут | Ошибка исполнения |

## 2.9. `НАПИШИ`

### Назначение

Единый оператор отправки сообщения. Покрывает и обычные сообщения, и запросы информации (ранее `СПРОСИ`).

### Каноническая форма

```coil
НАПИШИ answer
КУДА #быстрые_вопросы
КОМУ @тех_аналитик
ЖДАТЬ ЛЮБОЙ
НЕ БОЛЕЕ 10м
<<
Проверь возможную причину сбоя и безопасный обход.
>>
КОНЕЦ
```

### Модификаторы

| ID | Ключевое слово (RU) | Назначение | Принимает | Обязательность |
|---|---|---|---|---|
| `Mod.To` | `КУДА` | Адрес канала или поста | Ссылка `#name` | Нет |
| `Mod.For` | `КОМУ` | Получатели | Список ссылок `@name` через `,` | Нет |
| `Mod.ReplyTo` | `ОТВЕТ НА` | Путь к сообщению (включая комментарий), на которое отвечаем | Ссылка `#name` | Нет |
| `Mod.Await` | `ЖДАТЬ` | Политика ожидания | `Pol.None`, `Pol.Any`, `Pol.All` | Нет |
| `Mod.Timeout` | `НЕ БОЛЕЕ` | Таймаут ожидания | Duration literal | Нет |

Анонимное шаблонное тело — собственно сообщение. Должно идти последним.

### Семантика

- Среда исполнения обязана доставить сообщение по указанному адресу.
- Политика `ЖДАТЬ` определяет, когда обещание `?name` считается завершённым. Это свойство конкретного сообщения, а не оператора `ЖДИ`.
- `НАПИШИ` допускается без имени результата; в этом случае обещание не создаётся.

### Результаты (если указано имя)

| Сущность | Ссылка | Когда доступен |
|---|---|---|
| Обещание | `?name` | Сразу после отправки |
| Значение | `$name` | После выполнения политики ожидания |

### Условия завершения

Определяются политикой `ЖДАТЬ`:

| Политика | Условие завершения |
|---|---|
| `НЕТ` | Сообщение отправлено |
| `ЛЮБОЙ` | Получен хотя бы один ответ |
| `ВСЕ` | Получены ответы от всех адресатов |

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Невозможность доставить сообщение | Ошибка исполнения |
| Таймаут ожидания | Ошибка исполнения |

### Пример: ответ на сообщение

Оригинальное сообщение расположено по адресу: `#запросы_клиентов/1234`.
Ответ будет размещен в виде комментария: `Ответ: ...` 

```coil
НАПИШИ
КУДА #запросы_клиентов/1234
КОМУ @клиент_а
<<
Ответ: $answer
>>
КОНЕЦ
```

## 2.10. `ЖДИ`

### Назначение

Точка синхронизации: приостанавливает текущий экземпляр протокола до получения результатов.

### Каноническая форма

```coil
ЖДИ
НА ?plan, ?answer
РЕЖИМ ВСЕ
НЕ БОЛЕЕ 10м
КОНЕЦ
```

### Модификаторы

| ID | Ключевое слово (RU) | Назначение | Принимает | Обязательность |
|---|---|---|---|---|
| `Mod.On` | `НА` | Ожидаемые обещания | Список ссылок `?name` через `,` | Да (в стабильной форме) |
| `Mod.Mode` | `РЕЖИМ` | Режим ожидания | `Pol.All` или `Pol.Any` | Нет (по умолчанию `Pol.All`) |
| `Mod.Timeout` | `НЕ БОЛЕЕ` | Таймаут | Duration literal | Нет |

### Семантика

- `ЖДИ` блокирует **текущий экземпляр протокола**, а не весь агент.
- После успешного ожидания соответствующие значения становятся доступны как `$name`.
- `ЖДИ` не открывает новый поток и не создаёт обещаний.

> Примечание: событийная форма `ЖДИ` для потоков описана в [07-streams.md](07-streams.md) и считается рабочей гипотезой.

### Условия завершения

| Режим | Условие завершения |
|---|---|
| `ВСЕ` | Все перечисленные обещания разрешены |
| `ЛЮБОЙ` | Хотя бы одно обещание разрешено |

### Ошибки

| Условие | Класс ошибки |
|---|---|
| Обещание не существует | Ошибка подготовки |
| Таймаут | Ошибка исполнения |

---

## 2.11. `ВЫХОД`

### Назначение

Явно завершает текущий экземпляр протокола.

### Каноническая форма

```coil
ВЫХОД
```

### Семантика

- Завершает выполнение протокола.
- Не отправляет сообщение.
- Не имеет аргументов, модификаторов и тела.
- Записывается одной строкой без `КОНЕЦ`.

### Правила валидности

- Аргументы или модификаторы после `ВЫХОД` считаются ошибкой подготовки.
